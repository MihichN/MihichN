## Operating Principles

- Design around real constraints, not ideal provider behavior.
- Keep critical business rules explicit and testable.
- Treat admin actions, retries, and partial failures as product risks.
- Separate public showcase code from private production logic and credentials.
- Own system behavior as a whole, not just individual services.
